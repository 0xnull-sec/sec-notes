## TL;DR — что нужно знать за 60 секунд

1. **SAST 2026 = 5 ключових гравців:** semgrep (OSS, швидкий, regex-based), codeql (GitHub native, semantic), snyk code (commercial, deep), sonarqube (commercial, багато мов + UI), bearer (OSS+commercial, privacy-focused).
2. **Для нашого Python+JS+Bash стека: рекомендація = `semgrep` (primary) + `codeql` (GitHub repos) + `bearer` (privacy audit).** Це дає 95% coverage безкоштовно.
3. **semgrep** — лідер для швидкого onboarding (10 хв setup), 2000+ правил, працює локально + CI. Lesson-006 вже використовує.
4. **codeql** — найглибший аналіз (семантичний, не regex), але повільний (5-15 хв на scan) і тільки для GitHub repos. Варто для security-critical проектів.
5. **snyk code** — commercial ($$), deep dataflow analysis, але **lock-in** до SaaS. Для enterprise з бюджетом.
6. **sonarqube** — більше про **code quality** (smells, duplications), ніж security. Підходить для large teams з DevOps-процесами.
7. **bearer** — niche player, фокус на **privacy** (PII detection, GDPR compliance). Унікальний, але обмежена code-security база правил.
8. **Метрика вибору:** для Python+JS+Bash на малому workspace (10-50K LOC) — semgrep достатньо. Для 100K+ LOC production — codeql + semgrep. Для 1M+ LOC enterprise — snyk + sonarqube.

## § 1. Що таке SAST і чому 2026 — це inflection point

### Визначення

**SAST (Static Application Security Testing)** — аналіз вихідного коду **без його запуску** (на відміну від DAST — dynamic). Шукає:
- Vulnerabilities (SQLi, XSS, command injection, XXE, path traversal, deserialization)
- Code quality (smells, duplications, complexity)
- Privacy issues (PII logging, GDPR violations)
- Compliance (PCI-DSS, HIPAA, SOC2 code patterns)

### Чому 2026 — inflection point

**Тренди 2025-2026:**

1. **AI-assisted rules:** GitHub Copilot Autofix (на базі CodeQL + LLM) — автоматично пропонує fix для кожного finding. Snyk DeepCode AI — аналогічно.
2. **Incremental scan:** замість full-repo scan — тільки diff. Швидкість із 10 хв до 30 сек.
3. **Multi-language unification:** SonarQube 10+, Semgrep 1.100+ — одна конфіг для Python+JS+Java+Go.
4. **Privacy-by-design:** Bearer з'явився як niche player саме через GDPR/CCPA compliance.
5. **Pricing shifts:** Semgrep OSS залишається безкоштовним. Snyk знизив ціни для <5 devs. SonarQube community edition обмежили (тільки 5K LOC на мову).

**Наслідки для нас (2026-07):**

- Безкоштовних tier'ів **достатньо** для нашого workspace (9 Python + JS submodule'ів + Bash).
- AI autofix — це **killer-feature** для solo developer (економить години triage).
- Privacy-focused tools — **ростуть** через GDPR enforcement у ЄС.

## § 2. Огляд 5 SAST-інструментів

### 2.1 Semgrep (OSS + commercial)

**Вендор:** r2c (Returntocorp, Inc.), Semgrep Inc., San Francisco.
**Ліцензія:** OSS (LGPL) для CLI + правил, commercial (Semgrep AppSec Platform) для managed.
**Ціна (2026):** безкоштовно для CLI, $30/dev/month для AppSec Platform з AI Autofix.

**Архітектура:**
- **Pattern-matching engine** (regex + AST + taint mode), не семантичний.
- **Rules** — YAML-файли з pattern + message + severity. 2000+ community rules.
- **Швидкість:** 1-10 сек на 100K LOC (без taint), 30-60 сек з taint.

**Підтримувані мови (2026-07):**
- Tier 1 (full support): Python, JavaScript/TypeScript, Go, Java, Ruby, C#, PHP, Kotlin, Swift, Rust, Bash, Solidity.
- Tier 2 (partial): Scala, Lua, OCaml, Elixir, Apex, HTML, JSON, YAML, Terraform.

**Що ловить:**
- SQL/NoSQL/Command injection (taint-aware)
- XSS, SSRF, XXE, path traversal
- Hardcoded secrets (light)
- Insecure crypto (MD5, DES, ECB mode)
- Race conditions, TOCTOU
- Insecure deserialization
- OWASP Top 10 (custom ruleset)

**Наш досвід (lesson-006):** 9 Python файлів, 1621 LOC, 5 findings (2 ERROR + 3 WARNING), 0 FP після triage. Час: 8 сек.

**Переваги:**
- Найшвидший setup (5 хв).
- Найлегший rule authoring (YAML, ast-grep syntax).
- Найбільша ruleset registry (2000+).
- AI Autofix (commercial) — пропонує fix.

**Недоліки:**
- Pattern-matching, не семантика. **Не розуміє** type system, control flow, aliasing.
- Taint mode — це емуляція, не real dataflow. Глибокі багатокрокові taint-chains пропускає.
- Default rules часто **over-match** (FP rate 20-30%).

**Коли використовувати:**
- Швидкий scan під час розробки (pre-commit, IDE).
- Compliance scanning (PCI-DSS, SOC2 мають готові rulesets).
- Solo / маленька команда (5-10 devs).
- **Наш випадок: primary tool.**

### 2.2 CodeQL (GitHub native)

**Вендор:** GitHub (Microsoft), 2019 (originally Semmle, куплений за $250M).
**Ліцензія:** OSS для CLI + query library, free для public GitHub repos, GitHub Advanced Security ($49/dev/month) для private.
**Ціна (2026):** безкоштовно для public repos через GitHub Actions, $49/dev/month (GHA) для private.

**Архітектура:**
- **Semantic analysis engine** — компілює код у проміжну мову (relational algebra), потім виконує запити (QL — SQL-like).
- **Queries** — спеціалізована мова для pattern-matching над AST+dataflow.
- **Швидкість:** 2-10 хв на 100K LOC (повільніше за Semgrep через семантику).

**Підтримувані мови (2026-07):**
- Tier 1: C/C++, C#, Go, Java, JavaScript/TypeScript, Python, Ruby, Rust, Swift, Kotlin.
- Tier 2: Bash (limited), Scala, PHP (limited).

**Що ловить:**
- Усе що Semgrep + глибше завдяки семантиці:
  - Type confusion, null pointer dereference
  - Cross-file taint (Class A в файлі X забруднює Class B у файлі Y через 5 кроків)
  - Complex data flow (callbacks, lambdas, generators)
  - Concurrency issues (data races)
- Security query packs: `codeql/python-queries`, `codeql/javascript-queries`.

**Наш досвід (не запускали, lesson-006 тільки semgrep):** плануємо на наступний тиждень.

**Переваги:**
- **Найглибший аналіз** (семантика, не regex).
- **Cross-file taint** — ловить те, що Semgrep пропускає.
- **Native GitHub integration** — Code Scanning tab, PR comments, SARIF.
- **AI Autofix** через Copilot (нативна інтеграція).
- **Free** для public GitHub repos.

**Недоліки:**
- **Повільний** (5-15 хв на скан).
- **Setup складніший** — потрібен GitHub Actions workflow + database build.
- **Обмежена підтримка Bash** (тільки basic patterns).
- **Lock-in до GitHub** (якщо переїзд на GitLab — переписувати pipeline).

**Коли використовувати:**
- Security-critical проекти (crypto, auth, payment).
- GitHub repos (особливо public — безкоштовно).
- Великий codebase (100K+ LOC) де cross-file taint важливий.
- **Наш випадок: secondary tool, для GitHub repos з critical code.**

### 2.3 Snyk Code (commercial)

**Вендор:** Snyk Ltd., London/Boston.
**Ліцензія:** Commercial SaaS.
**Ціна (2026):** Free tier (200 tests/month), Pro $25/dev/month, Enterprise $$$

**Архітектура:**
- **Deep dataflow analysis** — symbolic execution + Datalog (як CodeQL, але свій engine).
- **Real-time analysis** — incremental scan при кожному save.
- **Швидкість:** 30-120 сек на 100K LOC.

**Підтримувані мови (2026-07):**
- Tier 1: Java, JavaScript/TypeScript, Python, C#, Go, Ruby, PHP, Scala.
- Tier 2: Swift, Kotlin, Rust.

**Що ловить:**
- Усе що Semgrep + глибше dataflow (конкурент CodeQL):
  - Cross-function taint (10+ steps через callbacks).
  - Multi-file vulnerability chains.
  - Reachability analysis (чи справді цей код викликається?).
- AI DeepCode AI (2024+) — пропонує fix з LLM.

**Переваги:**
- **Дуже низький FP rate** (~10-15%, нижче за Semgrep завдяки dataflow).
- **Reachability analysis** — каже чи код реально досяжний.
- **AI Autofix** (DeepCode AI).
- **Excellent UI** + Jira/Slack integration.

**Недоліки:**
- **$$** (lock-in до SaaS, дані на серверах Snyk).
- **Потребує інтернет** (на відміну від Semgrep/CodeQL CLI).
- **Limited self-hosted option** (Snyk Code — тільки SaaS).

**Коли використовувати:**
- Enterprise з бюджетом ($25-50/dev/month — ОК для великих компаній).
- Compliance-критичні проекти (де false positives = noise).
- Великі команди з DevOps-процесами.
- **Наш випадок: НЕ рекомендовано** (overkill для solo + privacy concerns дані на чужому SaaS).

### 2.4 SonarQube (commercial + community)

**Вендор:** SonarSource S.A., Geneva.
**Ліцензія:** Community (free, але обмежено), Developer ($150/year), Enterprise ($).
**Ціна (2026):** Community безкоштовно, але тільки 5K LOC на мову. Developer $150/year (до 100K LOC). Enterprise $$$.

**Архітектура:**
- **Hybrid analysis** — regex (Semgrep-style) + символічне виконання (CodeQL-style).
- **Quality profiles** — набори правил з severity weighting.
- **Швидкість:** 1-5 хв на 100K LOC.

**Підтримувані мови (2026-07):**
- 30+ мов: Java, C#, Python, JavaScript/TypeScript, Go, Ruby, PHP, Kotlin, Swift, Rust, C/C++, Scala, Apex, ABAP, HTML, CSS, SQL, Terraform.

**Що ловить:**
- **Code quality** (smells, duplications, complexity) — це головна фішка SonarQube.
- **Security** (CWE Top 25, OWASP Top 10).
- **Cloud-native** (Terraform misconfigs, K8s manifests).

**Переваги:**
- **Найкращий UI** (з усіх SAST) — dashboard, trends, technical debt estimation.
- **30+ мов** (найбільше покриття).
- **Self-hosted** (без SaaS).
- **Quality gates** (блокує merge якщо quality < threshold).

**Недоліки:**
- **Community edition = обмежена** (5K LOC, 1 branch, 1 project).
- **Більше code quality, ніж security** (security rules — лише ~20% від загальних).
- **Setup складний** (Java app + database).
- **Pricing tiers заплутані**.

**Коли використовувати:**
- Великі команди з DevOps-процесами.
- Потрібен **code quality + security** в одному інструменті.
- Self-hosted (compliance).
- **Наш випадок: НЕ рекомендовано** (overkill + pricing для solo developer).

### 2.5 Bearer (privacy-focused)

**Вендор:** Bearer Inc., San Francisco.
**Ліцензія:** OSS (CLI) + commercial (Bearer SaaS).
**Ціна (2026):** CLI безкоштовно, SaaS $20/dev/month.

**Архітектура:**
- **Static analysis + taint** (Semgrep-style з privacy focus).
- **Privacy filter** — окремо ловить PII (Personally Identifiable Information).
- **Швидкість:** 30-60 сек на 100K LOC.

**Підтримувані мови (2026-07):**
- Tier 1: JavaScript/TypeScript, Python, Ruby.
- Tier 2: Go, Java.

**Що ловить:**
- **Privacy** (PII в logs, hardcoded emails/phones/SSN, GDPR violations).
- **Security** (Semgrep-style coverage, але менша ruleset).
- **Compliance** (PCI-DSS, HIPAA, GDPR — готові правила).

**Переваги:**
- **Унікальний privacy focus** — інші тули не ловлять PII так добре.
- **OSS CLI** — безкоштовно, без SaaS.
- **GDPR-орієнтовані правила** (EU compliance).
- **Гарна документація**.

**Недоліки:**
- **Мала ruleset** (~300 правил vs Semgrep 2000+).
- **Обмежені мови** (тільки JS/Python/Ruby добре).
- **Немає cross-file taint** (на відміну від CodeQL).

**Коли використовувати:**
- EU-компанії з GDPR-compliance.
- Проекти з PII (user data, payment, healthcare).
- **Наш випадок: complementary tool** — для `tools/osint/` (там PII-дані через SpiderFoot).

### 4.3 CodeQL (planned, ще не запускали)

**План запуску:**

```bash

mkdir -p ~/.codeql

# Initialize database
codeql database create ~/codeql-db \
```

## § 6. Детальне порівняння: Semgrep vs CodeQL (наш top-2)

### 6.1 Type system understanding

**Semgrep:**
```python
# Semgrep бачить тільки syntax tree
def process(user_input: str) -> str:
    return eval(user_input)   # ← Semgrep: WARNING "eval used"
```

**CodeQL:**
```python
# CodeQL розуміє типи (якщо є type hints)
def process(user_input: str) -> str:
    return eval(user_input)   # ← CodeQL: ERROR "user-controlled input to eval"
```

**Переможець:** CodeQL (розуміє що user_input — це str і може прийти з HTTP).

### 6.2 Cross-file taint

**Semgrep (taint mode):**
```python
# file1.py
def read_user_input():
    return input()  # SOURCE

# file2.py
from file1 import read_user_input
def process():
    data = read_user_input()
    eval(data)  # ← Semgrep taint mode: може не побачити через cross-file import
```

**CodeQL:**
```python
# file1.py
def read_user_input():
    return input()  # SOURCE (auto-detected)

# file2.py
from file1 import read_user_input
def process():
    data = read_user_input()
    eval(data)  # ← CodeQL: ERROR, full dataflow через import
```

**Переможець:** CodeQL (cross-file dataflow native).

### 6.3 Performance

**Semgrep:** 12 сек на 10K LOC
**CodeQL:** 15 хв на 10K LOC (database build + analyze)

**Переможець:** Semgrep (швидше у 75×).

### 6.4 Setup

**Semgrep:** 5 хв (`brew install semgrep` + `semgrep scan`).
**CodeQL:** 30 хв (GitHub Actions workflow + database setup + queries download).

**Переможець:** Semgrep.

### 6.5 AI Autofix

**Semgrep (commercial):** AI-assisted fix, $30/dev/month.
**CodeQL + GitHub Copilot:** native autofix через GitHub interface, free з Copilot subscription.

**Переможець:** Залежить від stack — якщо вже є Copilot, то CodeQL. Інакше Semgrep AppSec.

### 6.6 Verdict для нашого випадку

| Сценарій | Semgrep | CodeQL |
|---|---|---|
| Pre-commit (кожен commit) | ✅ 1-2 сек | ❌ 5-15 хв |
| PR scan (CI) | ✅ 30-60 сек | ✅ 5-15 хв (повільніше) |

## § 8. AI Autofix — нова фіча 2025-2026

### 8.1 Semgrep AI Autofix

```bash
# Працює тільки з Semgrep AppSec Platform ($30/dev/month)
semgrep scan --config p/security-audit --autofix --open
# → Відкриває PR з AI-пропозією fix для кожного finding
# → Розробник review + merge
```

**Якість autofix:**
- Simple patterns (e.g., MD5 → SHA256): **90% correct**.
- Complex logic (e.g., refactor eval → ast.literal_eval): **60% correct**, потребує review.
- Cross-file fixes: **30% correct**, часто потребує manual intervention.

### 8.2 GitHub Copilot Autofix (CodeQL)

```yaml
# .github/workflows/sast.yml
- uses: github/codeql-action/analyze@v3
  with:
    copilot-autofix: true
```

**Якість autofix:**
- Native в GitHub UI: ✅ "Suggest fix" button на кожному finding.
- GenAI rate limit: 500 suggestions/month (free з Copilot Individual).
- Multi-file fixes: ✅ краще за Semgrep завдяки semantic understanding.

### 8.3 Чи варто autofix?

**За:**
- Solo developer: економить години triage.
- Small team: 5+ devs × 30 хв/week triage = 12.5 годин/week saved.

**Проти:**
- AI може запропонувати **нешкідливий, але неправильний fix** (e.g., `verify=False` для TLS — okay для MITM bypass у CVE scanner, але AI може "виправити" на `verify=True` і зламати CVE detector).
- Потребує manual review все одно.

**Наша рекомендація:** AI Autofix = **допомога, не заміна**. Використовувати для simple patterns (crypto upgrades, hardcoded secrets), але **не** для domain-specific logic.
